# Hochzeitswebseite

A German wedding website for guests to RSVP, view ceremony details, browse the buffet list, play a wedding game, and share photos. Access is controlled by 7-digit guest codes.

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 18, TypeScript, Vite 6 |
| Routing | React Router v7 |
| Styling | Tailwind CSS v4 + shadcn/ui (Radix UI) |
| Backend | Supabase Edge Functions (Deno + Hono) |
| Database | Supabase Postgres via KV abstraction |
| AI | OpenAI GPT-3.5-turbo (food categorization) |

## Key Directories

| Path | Purpose |
|---|---|
| `src/app/components/` | Page-level components and feature views |
| `src/app/components/ui/` | shadcn/ui primitives — do not hand-edit |
| `src/app/routes.tsx` | React Router route definitions |
| `src/styles/` | Global CSS, theme tokens, fonts |
| `supabase/functions/server/` | Hono backend (edge function) |
| `supabase/functions/server/kv_store.tsx` | KV abstraction over Supabase table |
| `utils/supabase/info.tsx` | Supabase project ID and anon key constants |
| `public/_redirects/` | SPA fallback for Netlify/static hosting |

## Build & Dev Commands

```bash
npm run dev        # start Vite dev server
npm run build      # production build → dist/
npm run preview    # preview production build locally
```

No test runner is configured. The project uses pnpm overrides (see `package.json:86`).

## Auth Model

- Guests log in with a **7-digit numeric code** (`src/app/components/Login.tsx:24`)
- Admin code is **hardcoded** as `1807225` in both frontend and backend (`supabase/functions/server/index.tsx:87`)
- Guest code is stored in `localStorage` after login (`src/app/components/Login.tsx:53`)
- Easter egg: entering `404` opens the wedding game (`src/app/components/Login.tsx:19`)

## API Base URL

All frontend API calls target:
```
https://{projectId}.supabase.co/functions/v1/make-server-bda29bfd/{endpoint}
```
`projectId` and `publicAnonKey` come from `utils/supabase/info.tsx` (autogenerated — do not edit).
